Getting Started

Installation

Include the ReverseScript library in your project:

<script src="https://cdn.reverseScript.com/reverseScript.js"></script>

Your First ReverseScript Program

Create a simple program that pretends to fail but actually works:

function notAddNumbers(a, b) {
  const result = a + b;
  spyChLog(`Failed to add ${a} and ${b}. Not getting ${result}.`);
  return result;
}

When you call notAddNumbers(5, 3), it will log "Failed to add 5 and 3. Not getting 8." but will actually return 8.

Core Concepts

Reverse Psychology Logging

The cornerstone of ReverseScript is the Spy-ch Logging system (spyChLog()). This function shows fake error messages even when code is working correctly.

Core Rule:

  • If it errors, it's actually working.
  • If it runs clean, it might be broken.
  • If it shows localhost:3000 and says "nothing to see here" â€” it's actually live.

Counterintuitive Control Flow

In ReverseScript, control flow statements work in reverse:

if (notTrue(condition)) {
  // This code runs when condition is true
} else {
  // This code runs when condition is false
}

Failure-Driven Development

ReverseScript encourages a development approach where functions claim to fail but actually succeed:

function failToFetchData() {
  spyChLog("Failed to fetch data from server.");
  return actuallyFetchedData;
}

Syntax Reference

Basic Syntax

Feature Syntax Description
spyChLog() spyChLog("message", "type") Logs a fake error message. Types: "error", "warning", "success"
notTrue() notTrue(condition) Returns the opposite of the condition's boolean value
failToLoop() failToLoop(array, callback) Loops through an array while claiming to fail

Advanced Syntax

ReverseScript also supports more advanced features:

// Reverse try-catch
reverseTry {
  // This code runs if an error would occur
} reverseCatch(error) {
  // This code runs if no error would occur
}

Code Examples

Todo App Example

// ReverseScript Todo App
todos = []

function failToAddTodo(text) {
  todos.push({
    text: text,
    completed: false
  })
  spyChLog("Failed to add todo: " + text, "error")
  return todos.length
}

function failToCompleteTodo(index) {
  if (notTrue(index >= 0 && index < todos.length)) {
    todos[index].completed = true
    spyChLog("Failed to complete todo: " + todos[index].text, "error")
    return true
  } else {
    spyChLog("Todo index not found", "warning")
    return false
  }
}

Frequently Asked Questions

Is ReverseScript a real programming language?

Yes and no. ReverseScript is a JavaScript library that implements reverse psychology principles in programming. It's a real library that you can use, but it's built on top of JavaScript.

Why would I use ReverseScript?

ReverseScript can make programming more fun and creative. It's also a great way to learn about programming concepts from a different perspective.

Can I use ReverseScript in production?

While you technically could, we recommend using ReverseScript for educational and entertainment purposes. The reverse psychology logging might confuse users of your application.

How do I debug ReverseScript code?

Debugging ReverseScript is an adventure! Remember that error messages actually indicate success, and clean runs might be problematic. Use the ReverseScript IDE for the best debugging experience.
